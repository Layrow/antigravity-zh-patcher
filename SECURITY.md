# 安全说明

本项目会修改本机已安装应用的资源文件，因此安全边界需要写清楚。

## 支持范围

请报告这些问题：

- 脚本可能误删或覆盖非 Antigravity 文件。
- 恢复流程无法恢复到备份。
- 安装脚本在应用运行时仍继续覆盖关键资源。
- 补丁逻辑可能导致凭据、个人文件或日志被意外打包进仓库。
- 文档中的命令存在明显破坏性风险。

## 不支持范围

- Antigravity 官方应用自身漏洞。
- Google 服务、账号、模型或后端行为问题。
- 修改后二进制签名失效带来的系统提示。

## 报告方式

如果仓库已经公开，可以优先开 GitHub issue，并避免贴出敏感路径、账号、token、日志或私有文件内容。

如果问题涉及真实凭据或数据泄漏，请先私下联系仓库维护者，再公开细节。

## 安全使用建议

- 运行安装脚本前，先确认官方应用可以正常启动。
- 覆盖资源前必须完全退出 Antigravity。
- 保留 `backups/`，确认新补丁稳定后再清理。
- 不要把 `backups/`、`dist/`、`extracted/`、`node_modules/` 推送到 GitHub。
